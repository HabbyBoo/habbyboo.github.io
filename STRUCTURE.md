# 🏪 Multi-Link Store Web Structure Specification
Version: v1.1
Project Type: GitHub Pages (Static Web App)
Purpose: 상품 링크를 효율적으로 관리하는 단일 페이지 기반 멀티링크 웹사이트

------------------------------------------------------------
📌 핵심 개념
------------------------------------------------------------
- 상품 UI는 JSON 기반 자동 렌더링
- 상품 추가 시 HTML/CSS/JS 수정 없음
- /db 디렉토리에 데이터/이미지 분리하여 유지보수 단순화
- 메인 페이지 중심 구조 (확장성 고려)

------------------------------------------------------------
📂 디렉토리 구조
------------------------------------------------------------
/ (repo root)
├─ index.html
├─ css/
│   style.css
├─ js/
│   render.js
├─ page1/
│   index.html
│   page1.css
└─ db/
    data/
        products.json
    img/
        item/
            0001.png
            0002.png
            ...
        tag/
            lowest.png
            psp.png
            netflix.png
            possible.png
            ...
        site/
            profile.png
            banner.png (optional)

------------------------------------------------------------
📦 products.json 데이터 구조
------------------------------------------------------------
[
  {
    "id": "0001",
    "display_name": "온열 망토",
    "link": "https://www.amazon.com/xxxx",
    "img": "0001.png",
    "tags": ["lowest"]
  },
  {
    "id": "0004",
    "display_name": "HCS350 Pro 게임 프로젝터",
    "link": "https://www.amazon.com/kkkk",
    "img": "0004.png",
    "tags": ["psp", "netflix", "possible"]
  }
]

필드 설명:
id → 상품 고유 ID (이미지 파일명과 동일하게 유지)
display_name → 사용자 UI에 표시되는 상품명
link → Amazon / 제휴 URL
img → /db/img/item/ 내부 이미지 파일명
tags → /db/img/tag/ 내부 태그 이미지 파일명 목록

상품 추가 방법:
1) products.json 에 엔트리 추가
2) /db/img/item/ 에 동일 id 이름의 이미지 추가
→ 자동 반영됨 (소스코드 수정 없음)

------------------------------------------------------------
🎨 UI 규칙
------------------------------------------------------------
프로필 이미지 (/site/profile.png) → 원형 표시
상품 이미지 (/item/*.png) → 둥근 사각형 (radius 12~18px)
태그 이미지 (/tag/*.png) → 정사각 + 작게 표시 + 모서리 둥글게
태그는 display_name 위에 가로로 나열

------------------------------------------------------------
📜 리스트 레이아웃
------------------------------------------------------------
- 세로 1열 스크롤형 UI
- 모바일 / 데스크탑 반응형 대응

------------------------------------------------------------
🔍 검색 기능
------------------------------------------------------------
검색창 입력값 → id + display_name 문자열 비교
포함 여부 기반 필터링
JSON 변경 시 자동 반영

------------------------------------------------------------
🚂 상단 안내문 (Scrolling Banner)
------------------------------------------------------------
초기 문구: 쿠팡_파트너스_안내_문구_testA
애니메이션: 오른쪽 → 왼쪽 반복 이동 (속도: 보통)

------------------------------------------------------------
🧱 확장성 (추후 페이지)
------------------------------------------------------------
추가 페이지 예시:
/page1/
    index.html
    page1.css

접속 경로:
https://username.github.io/page1/   (index.html 자동 인식)

공통 CSS 사용 원칙:
- 모든 페이지는 css/style.css 를 기본 공유
- 개별 페이지가 추가 UI가 필요할 경우 page1.css 로 보완 (선택 사항)

------------------------------------------------------------
✅ 유지보수 요약
------------------------------------------------------------
상품 추가/변경      → products.json + /db/img/item/           (코드 수정 불필요)
태그 추가           → /db/img/tag/ + products.json tags 항목   (코드 수정 불필요)
프로필 변경         → /db/img/site/profile.png                (코드 수정 불필요)
상단 문구 변경       → index.html                               (문구만 변경)
스타일 변경         → style.css                                (UI 전반 제어)
개별 페이지 스타일   → page1/page1.css                          (필요 시에만 사용)

------------------------------------------------------------
